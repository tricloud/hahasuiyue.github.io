接入说明

一、Android

要求应用app必须具有以下权限

<uses-permission android:name="android.permission.INTERNET" />

1、引入库文件

a、Eclipse

(1) 将 AAR 文件后缀名改为 ZIP 并解压

(2) 在解压得到的目录里创建一个 libs 文件夹

(3) 删除 armeabi、armeabi-v7a、arm64-v8a、x86、x86_64文件夹和R.txt文件

(4) 将classes.jar移动到libs文件夹下

(5) 将jni下和项目架构相同的文件夹(比如项目用到了armeabi和armeabi-v7a，则只拷贝armeabi和armeabi-v7a，其他的不要动)移到 libs 文件夹下

(6) 删除jni文件夹

(7) 在解压得到的目录里创建 project.properties 文件，内容如下：

target=android-14
android.library=true

(8) 导入sdk到eclipse中

(9) 按下述说明修改代码

b、Android Studio

假设原module名为app，新增的module名为netshield

(1) 点击 File > New Module

(2) 依次点击 Import .JAR/.AAR Package 和 Next

(3) 输入 AAR 或 JAR 文件的位置，然后点击 Finish

(4) settings.gradle文件中添加模块

include ':app', ': netshield'

(5) 编辑原module的build.gradle文件

implementation fileTree(dir: '../netshield', include: ['*.aar'])

  2. 在onCreate函数开头新增以下代码

    //Netshield.init()如果返回false表示SDK启动失败，此时应用需要退出
    if (false == Netshield.init()) {  
        //此处为举例，建议当init函数返回false后，弹窗进行提示，应用退出。
      Toast.makeText(this, "netshield init failed.", Toast.LENGTH_LONG).show();
      System.exit(-1);
    }

  3. 在onResume函数返回前新增以下代码

    Netshield.reinit();

二、IOS

(1) 添加netshield.framework到工程中

(2) Build Settings/Search Paths/Header Search Paths中添加$(PROJECT_DIR)/sdk/netshield.framework/Headers

(3) General/Embedded Binaries中添加netshield.framework

(4) 在应用进行网络连接前进行函数调用 

#import "netshield.h" //需要引入的头文件
​
@implementation AppDelegate
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    if(!secure_init()) {
        return NO;
    }
    return YES;
}
​
- (void)applicationDidEnterBackground:(UIApplication *)application {
    secure_background();
}
​
- (void)applicationDidBecomeActive:(UIApplication *)application {
    secure_reinit();
}
​
@end
